## Context

Создаётся игра Arkanoid с нуля на Python/Pygame. Проект задуман как учебный — мы пишем игровой цикл, физику столкновений, систему уровней сами, без готового движка. Сборка в один исполняемый файл.

## Goals / Non-Goals

**Goals:**
- Полностью рабочая игра Arkanoid с меню, игрой, game over
- Физика столкновений с углом отскока (не просто инверсия скоростей)
- Система уровней на YAML/JSON-конфигах
- Локальные рекорды (JSON)
- Сборка в .exe / Linux binary одной командой
- Чистая архитектура с отделённой игровой логикой от рендеринга
- Unit-тесты на core-логику

**Non-Goals:**
- Звук, музыка (добавляется позже при желании)
- Сетевая игра
- Сервер рекордов (отдельный проект)
- Редактор уровней (уровни прописываются вручную в конфигах)

## Decisions

### Архитектура: Entity-Component, упрощённая

```
┌──────────────────────────────────────────────────────┐
│                    main.py                            │
│  Game (состояния: menu, playing, paused, gameover)    │
├──────────────────────────────────────────────────────┤
│                                                        │
│  ┌──────────────┐   ┌──────────────┐   ┌────────────┐ │
│  │    Core      │   │   Render     │   │  Input     │ │
│  │  (entities,  │   │  (pygame     │   │  (pygame   │ │
│  │   physics,   │   │   draw)      │   │   events)  │ │
│  │   collisions)│   └──────────────┘   └────────────┘ │
│  └──────────────┘                                      │
│         │                                              │
│  ┌──────▼──────┐   ┌──────────────┐   ┌────────────┐ │
│  │   config    │   │ persistence  │   │  levels    │ │
│  │  (game.yaml)│   │ (leaderboard)│   │ (layouts)  │ │
│  └─────────────┘   └──────────────┘   └────────────┘ │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**Решение**: core-модуль **не импортирует pygame**. Вся игровая логика (коллизии, движение шарика, подсчёт очков) — чистый Python. Это даёт тестируемость и возможность переиспользования.

### Физика: полу-детерминированная

- **Шарик**: позиция `(x, y)`, скорость `(dx, dy)`, обновляется каждый кадр
- **Отскок от платформы**: угол зависит от точки удара относительно центра платформы
  ```
       ┌─────────────────┐
       │   платформа      │
  -45° │-30°│ -15°│ 15°│30°│ 45°
       └─────────────────┘
  ```
  Самый край → максимальный угол (45°), центр → по вертикали вверх
- **Отскок от стен**: просто инверсия dx
- **Отскок от кирпича**: определение стороны столкновения (AABB), инверсия соответствующей компоненты скорости

### Кирпичи: типы через компонент здоровья

```python
class Brick:
    x, y, w, h      # позиция и размер
    hp: int         # 1 = обычный, 2+ = прочный
    max_hp: int
    bonus_type: str | None  # какой бонус выпадает при разрушении
```

Визуализация: цвет меняется от hp (тёмный = целый, светлый/треснувший = повреждённый).

### Уровни: конфиг-файлы

```yaml
# levels/level_01.yaml
name: "First Steps"
speed: 1.0           # множитель скорости шара
paddle_width: 120    # кастомная ширина платформы
bricks:
  - row: 0           # строка сверху
    cols: 0..12      # столбцы
    type: normal     # hp=1
  - row: 1
    cols: 0..12
    type: strong     # hp=2
    color: [100, 200, 100]
```

### Бонусы

Выпадающие предметы из кирпичей определённого типа. Падают вниз. При касании платформой — активируются на таймер:

| Бонус | Эффект | Длительность |
|-------|--------|-------------|
| 🟦 Wide | платформа ×1.5 | 10 сек |
| 🟥 Slow | скорость шара ×0.7 | 8 сек |
| 🟨 Multi | +2 шара | до потери всех |
| 🟪 Sticky | шар прилипает к платформе | 1 раз |

## Risks / Trade-offs

- **[Риск] Физика не идеальна** — возможны баги с "застреванием" шара в кирпиче на высоких скоростях
  → **Митигация**: подменять позицию шара при обнаружении пересечения, не полагаться на дельту времени
- **[Риск] PyInstaller .exe на Linux → Windows** — кросс-компиляция проблематична
  → **Митигация**: собираем на каждой платформе отдельно; GitHub Actions для CI
- **[Риск] Разрастание объёма** — легко увлечься бонусами и эффектами
  → **Митигация**: roadmap зафиксирован в proposal, MVP строго ограничен
