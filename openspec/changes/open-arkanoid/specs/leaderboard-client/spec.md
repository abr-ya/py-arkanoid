## ADDED Requirements

### Requirement: Клиент рекордов умеет отправлять данные на сервер

Система SHALL иметь класс `LeaderboardClient`, который:
- Сохраняет рекорды локально (обязательно)
- Пытается отправить их на удалённый сервер (опционально)
- Не прерывает игру при ошибке сети

#### Scenario: Сервер доступен
- **WHEN** после завершения игры сервер рекордов отвечает на HTTP-запрос
- **THEN** рекорд отправляется на сервер, ответ сервера логируется

#### Scenario: Сервер недоступен
- **WHEN** сервер рекордов не отвечает (таймаут, connection refused)
- **THEN** рекорд НЕ блокирует игру, сохраняется только локально

#### Scenario: Сервер конфигурируется
- **WHEN** приложение запускается
- **THEN** адрес сервера задаётся в переменной окружения `LEADERBOARD_URL` или в конфиг-файле `leaderboard_config.yaml`, значение по умолчанию — `http://localhost:8000`

### Requirement: Клиент предоставляет REST API для локального и удалённого режима

Система SHALL иметь функции для:
- `submit_score(name, score)` — отправка рекорда
- `get_top(limit=10)` — получение топ-10

#### Scenario: Запрос топ-10
- **WHEN** вызывается `get_top(10)`
- **THEN** возвращается список из 10 (или меньше) лучших записей
