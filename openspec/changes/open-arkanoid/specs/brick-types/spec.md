## ADDED Requirements

### Requirement: Кирпич имеет здоровье

Каждый кирпич SHALL иметь атрибут `hp` (int, >= 1). При ударе шара `hp -= 1`. При `hp <= 0` кирпич считается разрушенным и удаляется. Если кирпич разрушен — может выпасть бонус.

#### Scenario: Обычный кирпич
- **WHEN** шар ударяет кирпич с `hp: 1`
- **THEN** `hp` становится 0, кирпич разрушается немедленно

#### Scenario: Прочный кирпич
- **WHEN** шар ударяет кирпич с `hp: 2`
- **THEN** `hp` уменьшается до 1, кирпич меняет внешний вид (трещина/бледнее)
- **WHEN** шар ударяет тот же кирпич снова
- **THEN** `hp` становится 0, кирпич разрушается

#### Scenario: Неуязвимый кирпич
- **WHEN** кирпич имеет `hp: -1` (специальный флаг)
- **THEN** он НЕ разрушается при ударах; используется для декоративных/препятствующих элементов

### Requirement: Кирпич имеет цветовую кодировку состояния

Система SHALL визуально показывать, сколько HP осталось у кирпича. Цвет кирпича меняется от тёмного (полный HP) к светлому (1 HP).

#### Scenario: Один HP
- **WHEN** кирпич имеет `hp: 1`
- **THEN** отрисовывается в ярком/светлом цвете

#### Scenario: Два и более HP
- **WHEN** кирпич имеет `hp >= 2`
- **THEN** отрисовывается в тёмном оттенке, с визуальной индикацией "толщины"

### Requirement: Кирпичи могут быть разных типов

Система SHALL поддерживать типы кирпичей, определённые в конфиге уровня:
- `normal` — hp: 1, без бонуса
- `strong` — hp: 2, без бонуса
- `bonus` — hp: 1, при разрушении выпадает бонус (тип бонуса задаётся в конфиге)
- `indestructible` — hp: -1 (нельзя разрушить)
- `extra_life` — hp: 1, даёт +1 жизнь при разрушении

#### Scenario: Разные типы на одном уровне
- **WHEN** уровень загружается
- **THEN** все кирпичи разных типов корректно отображаются и обрабатываются согласно своему типу
