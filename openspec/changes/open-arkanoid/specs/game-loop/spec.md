## ADDED Requirements

### Requirement: Game управляет состояниями

Система SHALL иметь конечный автомат состояний: `MENU`, `PLAYING`, `PAUSED`, `GAME_OVER`, `EXIT`. Переходы между состояниями обрабатываются централизованно в Game-классе.

#### Scenario: Запуск в MENU
- **WHEN** игра запускается
- **THEN** начальное состояние — `MENU`

#### Scenario: Начало игры из MENU
- **WHEN** игрок нажимает ENTER/SPACE в MENU
- **THEN** состояние переключается на `PLAYING`, начинается загрузка уровня

#### Scenario: Пауза во время игры
- **WHEN** игрок нажимает ESC во время `PLAYING`
- **THEN** состояние переключается на `PAUSED`

#### Scenario: Снятие паузы
- **WHEN** игрок нажимает ESC (или ту же клавишу) в `PAUSED`
- **THEN** состояние возвращается в `PLAYING`

#### Scenario: Game Over
- **WHEN** все жизни потеряны ИЛИ последний шар улетел за нижнюю границу
- **THEN** состояние переключается на `GAME_OVER`

#### Scenario: Рестарт из GAME_OVER
- **WHEN** игрок нажимает ENTER/SPACE в `GAME_OVER`
- **THEN** состояние возвращается в `PLAYING` с перезагрузкой текущего уровня

#### Scenario: Выход в MENU из GAME_OVER
- **WHEN** игрок нажимает ESC в `GAME_OVER`
- **THEN** состояние переключается на `MENU`

### Requirement: Game-loop работает на фиксированной или переменной частоте кадров

Система SHALL обновлять состояние игры каждый кадр через `update(dt)` и отрисовывать через `draw(screen)`. `dt` — дельта времени в миллисекундах с предыдущего кадра. FPS ограничен на уровне 60 (pygame.Clock.tick(60)).

#### Scenario: Стабильный FPS
- **WHEN** игра работает при нормальной нагрузке
- **THEN** FPS = 60 ± 5
