## ADDED Requirements

### Requirement: Рекорды сохраняются в локальный JSON-файл

Система SHALL сохранять таблицу рекордов в файл `leaderboard.json` в корневой папке игры (рядом с исполняемым файлом). Формат:

```json
{
  "records": [
    {"name": "Игрок", "score": 15000, "date": "2026-07-03T12:00:00"}
  ]
}
```

#### Scenario: Запись после Game Over
- **WHEN** игра завершена и счёт > 0
- **THEN** система предлагает ввести имя (3 символа, по умолчанию "AAA")
- **WHEN** имя введено
- **THEN** запись добавляется в leaderboard.json

#### Scenario: Пустой рекорд
- **WHEN** рекордов ещё нет
- **THEN** файл содержит пустой массив `{"records": []}`

#### Scenario: Коррупция файла
- **WHEN** leaderboard.json повреждён или имеет невалидный формат
- **THEN** система игнорирует файл и начинает с пустого списка, без потери игры

### Requirement: Топ-10 рекордов отображается на экране

Система SHALL показывать таблицу лучших рекордов на экране GAME_OVER. Рекорды сортируются по убыванию очков, показываются только первые 10.

#### Scenario: Показ рекордов
- **WHEN** экран GAME_OVER активен
- **THEN** рядом с итоговым счётом показывается таблица "Top Scores" с первыми 10 записями

#### Scenario: Меньше 10 рекордов
- **WHEN** в таблице < 10 записей
- **THEN** все записи отображаются, пустые слоты не показываются
