## Why

Создать игру Arkanoid как учебный проект на Python с Pygame. Игра должна быть простой в сборке (один .exe-файл для Windows, запуск без зависимостей на Linux), чтобы её можно было легко показать и поделиться. В будущем — сохранение рекордов через отдельный сервер (FastAPI), но MVP полностью локальный.

Ключевая мотивация: **понимать, что происходит**. Не брать готовый движок, а писать игровой цикл, физику столкновений, систему уровней самому. Python + Pygame даёт ровно это.

## What Changes

- **Новая игра**: Arkanoid на Python/Pygame с классической механикой
- **Очки**: подсчёт за разбитые кирпичи, отображение на экране
- **Уровни сложности**: прогрессия через скорость шара, типы кирпичей, формации
- **Рекорды**: сначала локально (JSON), потом отправка на сервер
- **Сборка**: `PyInstaller --onefile --windowed` для Windows, `chmod +x`-совместимый запуск на Linux
- **Сервер рекордов**: отдельный проект, не в этом скоупе, но клиент готов к интеграции
- **Тесты**: базовые unit-тесты на игровую логику (коллизии, очки, состояния)

## Capabilities

### New Capabilities

- **game-loop**: Основной игровой цикл Arkanoid — инициализация, события, обновление, отрисовка. Владеет FPS и состояниями (меню, игра, пауза, game over).
- **physics**: Физика столкновений шара с платформой, кирпичами, стенами. Угол отскока от платформы (зависит от точки удара).
- **brick-types**: Типы кирпичей — обычные (1 удар), прочные (N ударов), неуязвимые (бонусные, неразрушаемые). HP кирпичей, визуальная индикация.
- **scoring**: Система очков — базовые очки за кирпич + бонусные множители (комбо за серию, очки за выживание).
- **levels**: Система уровней — загрузка формаций кирпичей из конфига. Прогрессия сложности (скорость шара, плотность кирпичей). Возможность прохождения уровня (все кирпичи разбиты).
- **bonuses**: Бонусные модификаторы — выпадающие улучшения из кирпичей: широкая платформа, мульти-шар, замедление, липкая платформа.
- **persistence**: Локальное сохранение рекордов в JSON-файл. Таблица лучших результатов с именем игрока.
- **leaderboard-client**: HTTP-клиент для отправки рекордов на внешний сервер. Заготовка, которая работает офлайн и переключается на онлайн при наличии сервера.
- **build**: Система сборки — скрипты PyInstaller для Windows, Nuitka/PyInstaller для Linux. Один артефакт — один исполняемый файл.

### Modified Capabilities

Нет. Это новый проект с нуля.

## Impact

- **Зависимости**: Python 3.10+, Pygame, PyInstaller (сборка), requests (для рекордов)
- **Время**: MVP за один подход, full scope — последовательно
- **Тесты**: pytest на core-логику (коллизии, очки, состояния)
